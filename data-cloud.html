Módulo 1

O que é o Salesforce Data Cloud?

O Salesforce Data Cloud é o Customer Data Platform (CDP) nativo do ecossistema Salesforce. Ele ingere dados de múltiplas fontes — CRM, web, mobile, e-commerce, sistemas legados — e os une em um perfil unificado do cliente (Unified Customer Profile).

Ao contrário de soluções de BI tradicionais, o Data Cloud é projetado para ativação em tempo real: os segmentos construídos no CDP podem ser imediatamente usados para personalizar experiências no MCP, disparar jornadas no Marketing Cloud ou atualizar registros no Sales Cloud.

💡
Data Cloud vs. MCP: O Data Cloud é o repositório e o motor de identidade. O MCP é o canal de personalização em tempo real. Os dois se complementam: Data Cloud constrói o perfil de longo prazo; MCP reage ao comportamento imediato.

Módulo 3

Identity Resolution — o coração do CDP

A Identity Resolution é o processo que transforma registros fragmentados em um perfil único e confiável. É o que permite que o Data Cloud saiba que o "João Silva" do CRM, o "jsilva@email.com" do email marketing e o cookie anônimo do site são a mesma pessoa.

Match Rules
Regras que definem como dois registros são considerados a mesma pessoa. Podem ser exatas (email igual) ou fuzzy (nome similar + CEP igual).
Reconciliation Rules
Quando dois registros fazem match, qual valor "vence"? A fonte mais recente? A mais confiável? Você define a prioridade por campo.
Individual vs. Contact
Individual é o perfil unificado. Cada Contact Point (email, telefone, cookie) é uma entrada que pode levar ao mesmo Individual.
Unified Individual
O objeto final após Identity Resolution. Concentra todos os atributos e histórico de interação do cliente em uma única entidade consultável.
⚠️
Atenção: Identity Resolution em excesso pode criar "super perfis" incorretos (quando pessoas diferentes são unidas por erro). Calibre as regras de match com rigor e monitore a taxa de merge regularmente.

Módulo 4

Data Cloud + MCP — a integração mais poderosa

A integração entre Data Cloud e MCP cria um loop de dados bidirecional que potencializa ambas as ferramentas:

Segmentos CDP → Regras MCP
Um segmento "Clientes VIP com churn risk" criado no Data Cloud pode ativar automaticamente uma campanha de retenção no MCP — sem nenhuma integração manual.
Atributos CDP → User Attributes MCP
Atributos calculados no CDP (LTV estimado, propensão a compra, segmento de produto favorito) chegam ao perfil do MCP e enriquecem as decisões de personalização.
Eventos MCP → Data Cloud
Cada interação rastreada pelo MCP (page view, click, add to cart) retroalimenta o CDP — enriquecendo o Unified Profile com comportamento digital em tempo real.

Módulo 5

Boas práticas em projetos Data Cloud

Defina o data model antes de ingerir dados
Mapear os DMOs e seus relacionamentos antes de conectar fontes evita retrabalho custoso. Um modelo mal pensado compromete toda a Identity Resolution.
Comece com menos fontes, vá adicionando
É tentador conectar tudo de uma vez. Mas validar a qualidade dos dados fonte a fonte garante que o Unified Profile seja confiável desde o início.
Governe o volume de dados
Data Cloud cobra por volume de dados processados. Filtre eventos irrelevantes no ingresso. Não armazene o que não será usado em segmentação ou ativação.
Documente as regras de match
As regras de Identity Resolution são decisões de negócio com impacto técnico. Documente cada regra, seu racional e monitore periodicamente a qualidade do match.
