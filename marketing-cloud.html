Módulo 1

O ecossistema Marketing Cloud

O Salesforce Marketing Cloud (SFMC) é a plataforma de automação de marketing do ecossistema Salesforce. Ele orquestra comunicações em múltiplos canais — email, SMS, push, social, web — com base em dados comportamentais e de CRM.

O SFMC é composto por vários studios e builders, cada um com uma função específica. Entender como eles se conectam é fundamental para construir jornadas eficazes e integradas.

📧
Email Studio
Criação, personalização e disparo de emails. Suporta conteúdo dinâmico, testes A/B e personalização via AMPscript e SSJS.
🗺️
Journey Builder
Orquestração de jornadas multicanal. Define o fluxo de comunicação com base em eventos, comportamentos e dados do cliente.
📊
Audience Builder
Segmentação de audiências com base em atributos e comportamentos de múltiplas fontes de dados do SFMC.
🔗
Contact Builder
Gerencia o modelo de dados do SFMC — Data Extensions, relacionamentos e a estrutura que suporta personalização em escala.
💬
Mobile Studio
SMS, push notifications e mensagens in-app. Permite comunicação em tempo real baseada em geolocalização e comportamento mobile.
⚙️
Automation Studio
Automação de processos de dados — importação, transformação, segmentação e disparo agendado. Complementa o Journey Builder.

Módulo 2

AMPscript — personalização dentro do email

O AMPscript é a linguagem de script do SFMC para personalização de conteúdo em emails, landing pages e SMS. Ele é processado server-side no momento do envio, permitindo lógica condicional, lookups em Data Extensions e até recomendações do MCP. 

%%[ /* Exemplo: personalização por perfil no email */

SET @email = AttributeValue('emailaddr')
SET @firstName = AttributeValue('First Name')
SET @profile = AttributeValue('Profile')

/* Link para landing page com identidade passada via parâmetro */
SET @landingUrl = CONCAT(
  'https://zavolidouglas.github.io/SalesTech-Academy/?email=',
  URLEncode(@email),
  '&profile=',
  @profile
)

/* Conteúdo condicional por perfil */
IF @profile == 'arquiteto' THEN
  SET @headline = 'Explore arquitetura MCP'
ELSEIF @profile == 'dev' THEN
  SET @headline = 'Acesse exemplos de Event API'
ELSE
  SET @headline = 'Continue sua jornada Salesforce'
ENDIF
]%%


<h1>Olá, %%=v(@firstName)=%%!</h1>
<p>%%=v(@headline)=%%</p>
<a href="%%=v(@landingUrl)=%%">Acessar plataforma</a>
💡
Integração MCP + Marketing Cloud: O link gerado pelo AMPscript pode carregar parâmetros de identificação que o MCP usa para unir o perfil do email ao comportamento web no momento do clique.

Módulo 3

Journey Builder — orquestrando jornadas

O Journey Builder permite criar fluxos de comunicação automatizados baseados em eventos e comportamentos. Uma jornada pode ser simples (email de boas-vindas) ou complexa (onboarding multicanal com ramificações por comportamento).

Entry Event
Formulário preenchido
→
Email 1
Boas-vindas + link
→
Abriu email?

→
Email 2
Conteúdo por perfil
→
Conversão
Goal achieved
Entry Sources — como os contatos entram na jornada
API Event (tempo real), Data Extension (batch), Salesforce Object, MobileConnect, CloudPages Form. Cada source tem características de latência e volume diferentes.
Decision Splits — ramificações inteligentes
Divide o fluxo baseado em atributos do contato, comportamento (abriu email? clicou?) ou dados externos via API. Permite jornadas altamente personalizadas por segmento.
Einstein STO — horário ideal de envio
Einstein calcula o melhor horário para enviar o email de cada contato individualmente, maximizando taxa de abertura com base no histórico de comportamento.

Módulo 4

SFMC + MCP — o loop de personalização completo

A integração entre Marketing Cloud e MCP cria o loop mais completo de personalização omnicanal da Salesforce:

Recomendações MCP dentro do email
O MCP disponibiliza recomendações personalizadas que podem ser inseridas diretamente no corpo do email, entregando o conteúdo certo para cada destinatário no momento do envio.
Comportamento no email enriquece o perfil
Aberturas e cliques rastreados pelo Marketing Cloud retroalimentam o perfil no MCP, tornando as próximas personalizações ainda mais precisas.
MCP aciona jornadas no Marketing Cloud
Comportamentos capturados pelo MCP — como visitas repetidas sem conversão — podem disparar automaticamente jornadas de recuperação no Journey Builder.
Identidade unificada entre canais
O perfil do usuário no MCP e no Marketing Cloud é unificado nativamente — cliques no email, comportamento web e dados de CRM convergem em uma visão única do cliente.

Módulos 5 a 8

Boas práticas, deliverability e escala

Deliverability — o maior vilão silencioso
Taxa de entrega depende de reputação de IP/domínio, qualidade da lista e conteúdo. Configure SPF, DKIM e DMARC corretamente. Faça IP warming antes de grandes disparos.
Data Extensions bem modeladas = jornadas escaláveis
Um modelo de dados mal projetado no Contact Builder cria gargalos de performance em jornadas com alto volume. Normalize os dados, evite DEs gigantes com muitas colunas.
Teste sempre antes de ativar
Use Test Send para emails, Test Mode no Journey Builder e valide AMPscript no Preview antes de qualquer disparo. Um erro em produção com lista grande é custoso.
Monitore Engagement Rate, não só OR/CTR
Open Rate perdeu confiabilidade com Apple MPP. Priorize métricas de clique, conversão e dados do MCP para medir real engajamento das campanhas.
