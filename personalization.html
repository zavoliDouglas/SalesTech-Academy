Módulo 1

O que é o Marketing Cloud Personalization?

O Marketing Cloud Personalization (MCP) — anteriormente conhecido como Interaction Studio e Evergage — é a solução de personalização em tempo real da Salesforce. Ele captura dados comportamentais de visitantes em canais digitais e usa esses dados para entregar experiências individualizadas instantaneamente.

O diferencial do MCP em relação a outras ferramentas de personalização está na sua capacidade de unificar identidade anônima e identificada no mesmo perfil, e tomar decisões de personalização em milissegundos — sem depender de batch ou processamento offline.

💡
Conceito-chave: O MCP trabalha com dois estados de usuário: anônimo (visitante sem identificação) e identificado (usuário com email ou ID conhecido). No momento da identificação, os perfis são mesclados automaticamente — preservando todo o histórico comportamental anônimo.

Módulo 2

Os 4 pilares do MCP

Toda implementação do MCP orbita em torno de quatro conceitos fundamentais. Entender cada um deles é o que separa uma implementação rasa de uma verdadeiramente eficaz.

🗺️
Sitemap
Define quais páginas e ações são rastreadas. É o mapa que diz ao MCP onde ele está e o que está acontecendo — página de produto, checkout, homepage, artigo.
📦
Catalog
O inventário de itens personalizáveis — produtos, artigos, cursos, ofertas. Alimenta os algoritmos de recomendação (Recipes) com metadados ricos.
👤
User Attributes
Atributos do perfil do usuário — comportamentais, declarativos e inferidos. São a matéria-prima para segmentação e personalização de conteúdo.
📡
Event API
API REST para enviar eventos de canais offline ou server-side — CRM, app mobile nativo, call center. Estende a coleta para além do web beacon.

Módulo 3

Web Campaigns & Templates

As Web Campaigns são o mecanismo de entrega de personalização no canal web. Cada campanha define: onde aparece (zona), quem vê (segmento/regra), o que aparece (experiência) e quando (schedule/trigger).

As experiências são renderizadas via Templates — componentes HTML/CSS/JS que o MCP injeta no DOM da página em tempo real. O template recebe o contexto do usuário e decide o que renderizar. 

// Exemplo de Template JS — Personalização de banner
template.bind(context => {
  const user = context.user;
  const firstName = user.attributes.'First Name' || 'Visitante';
  const profile = user.attributes.'profile' || 'generic';

  // Conteúdo muda conforme o perfil
  const content = {
    arquiteto: 'Explore arquitetura de soluções MCP',
    dev: 'Acesse a Event API e exemplos de código',
    admin: 'Configure sua primeira Web Campaign'
  };

  return {
    headline: `Olá, ${firstName}!`,
    message: content[profile] || content.admin
  };
});
🎯
Boas práticas: Sempre defina uma experiência de controle (sem personalização) para seus testes A/B. O MCP distribui automaticamente o tráfego entre variantes e reporta lift estatístico.

Módulo 4

Recipes — Algoritmos de Recomendação

Os Recipes são os algoritmos de recomendação do MCP. Eles combinam sinais comportamentais do usuário com metadados do Catalog para sugerir os itens mais relevantes em cada momento.

Collaborative Filtering
Recomenda itens que usuários com comportamento similar consumiram. Ideal para "Quem viu isso também viu..."
Content Affinity
Analisa os atributos dos itens visualizados e recomenda itens com atributos similares. Perfeito para recomendações por categoria ou tema.
Trending & Popular
Recomenda itens com alta taxa de engajamento no momento. Útil para usuários novos sem histórico suficiente (cold start).
Einstein Decisions
Usa machine learning para escolher automaticamente o melhor algoritmo por usuário. Requer volume mínimo de dados para ativação.
⚠️
Cold Start: Recipes baseados em comportamento precisam de dados históricos para funcionar bem. Para novos usuários ou catálogos pequenos, combine com regras de negócio (boosting/blocking) para garantir relevância desde o primeiro acesso.

Módulo 6

Boas práticas e lições de projetos reais

Comece pelo Sitemap, não pelas campanhas
Um Sitemap mal configurado invalida toda a estratégia de personalização. Antes de criar qualquer campanha, mapeie exaustivamente cada tipo de página e ação relevante do site.
Nomeie eventos e atributos com convenção clara
Use snake_case, prefixos por área de negócio e documentação centralizada. Nomes como "btn_click" são inúteis em 6 meses. Prefira "checkout_add_to_cart_click".
Sempre tenha uma experiência de controle
Nunca publique uma campanha sem variante de controle. Sem isso, você não consegue medir o impacto real da personalização e justificar o investimento.
Respeite o volume mínimo para ML
Einstein Decisions e Collaborative Filtering precisam de dados. Em catálogos pequenos (<100 itens) ou sites novos, regras determinísticas superam ML na maioria dos casos.
Cuidado com o "over-personalization"
Personalização excessiva pode parecer invasiva. Mantenha um equilíbrio — nem todo elemento da página precisa ser dinâmico. Personalize onde o contexto do usuário realmente importa.
